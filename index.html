Comandos Git e Uso no Projeto

Configuração e Acesso ao Repositório GitHub

Para configurar o Git com seu usuário GitHub, siga os passos abaixo:

git config --global user.name "Seu Nome"
git config --global user.email "seuemail@example.com"
git config --global init.defaultBranch main

Para clonar um repositório GitHub:

git clone https://github.com/usuario/repositorio.git

Comandos Git

git init

• Inicializa um novo repositório Git em um diretório existente. Cria um subdiretório .git que contém todos os arquivos necessários do repositório - um esqueleto de um novo repositório Git.

git init

git clone

• Cria uma cópia (clone) de um repositório Git existente em um novo diretório. Ele também configura o repositório remoto original como origin.

git clone https://github.com/usuario/repositorio.git

git branch

Gera, lista ou deleta branches no repositório. Branches permitem que você trabalhe em diferentes versões de um projeto simultaneamente.

git branch

Para criar uma nova branch:

git branch nome-da-branch

git checkout

• Troca entre branches ou restaura arquivos na árvore de trabalho. Pode ser usado para criar uma nova branch e trocá-la ao mesmo tempo.

git checkout nome-da-branch

git status

Mostra o estado dos arquivos no repositório.

git status

git diff

• • Mostra as diferenças entre commits, branches, arquivos no índice de staging e no diretório de trabalho. Útil para ver o que mudou antes de fazer um commit.

git diff

git add

Adiciona alterações no diretório de trabalho ao índice de staging, preparando-as para serem incluídas no próximo commit. Pode adicionar arquivos específicos ou todas as alterações.

git add arquivo.txt

Para adicionar todos os arquivos modificados:

git add .

git commit

Grava as mudanças no índice de staging em um novo commit. É necessário fornecer uma mensagem de commit que descreve as mudanças feitas.

git commit -m "Mensagem do commit"

git push

• Envia commits do repositório local para um repositório remoto. Atualiza o repositório remoto com as alterações feitas localmente.

git push origin nome-da-branch

git pull

• Baixa commits de um repositório remoto e os integra ao branch atual. É uma combinação de git fetch e git merge.

git pull origin nome-da-branch

git revert

• Cria um novo commit que desfaz as mudanças de um commit anterior. É utilizado para desfazer mudanças sem alterar o histórico de commits.

git revert id-do-commit

git merge

• Combina alterações de diferentes branches em uma única branch. É usado para integrar mudanças de uma branch (por exemplo, uma branch de feature) de volta à branch principal.

git merge nome-da-branch

git stash

• Armazena temporariamente mudanças não commitadas para que você possa voltar a um estado limpo de trabalho. Pode ser utilizado para salvar mudanças sem fazer um commit.

git stash

Para aplicar mudanças armazenadas:

git stash apply

git rm

Remove arquivos do repositório.

git rm arquivo.txt

git config

• Define configurações de usuário e opções de repositório. Pode ser utilizado para configurar opções globais ou específicas de um repositório.

git config --global user.name "Seu Nome"

git reset

Reseta mudanças no índice e na árvore de trabalho.

git reset --hard

git remote

Gerencia repositórios remotos. Pode ser usado para adicionar, listar e remover repositórios remotos.

git remote add origin https://github.com/usuario/repositorio.git

git fetch

• Baixa objetos e refs de um repositório remoto sem integrá-los à branch atual. É usado para atualizar o repositório local com as mudanças do repositório remoto.

git fetch origin

git show

Mostra detalhes sobre um objeto Git (commit, tag, etc.).

git show id-do-commit

git help

Mostra ajuda sobre comandos Git.

git help comando

git rebase

Reaplica commits em cima de outra base.

git rebase nome-da-branch

git blame

Mostra o autor de cada linha de um arquivo.

git blame arquivo.txt

git tag

Marca commits com uma tag específica.

git tag -a v1.0 -m "Versão 1.0"
